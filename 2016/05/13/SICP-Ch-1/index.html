$(5+4+(2-(3-(6+4/5))))/(3(6-2)(2-7))$

(/ (+ 5 4 (- 2 (- 3 (+ 6 (/ 4 5)))))
  (* 3 (- 6 2) (- 2 7)))

运行结果

Exercise 1.3

Define a procedure that takes three numbers as arguments and returns the sum of the squares of the two larger numbers.

解答:

这道题主要的是如何找到三个数中较大的两个数，假设这三个数是a,b,c，定义bigger 和 smaller 分别返回两个数中较大的以及较小的数。具体算法如下：

  1. (bigger a b) 将会返回三个数中至少第二大的数。
  2. (bigger (smaller a b) c) a,b 中较小的数和c相比，返回较大的数，这个数为另一个至少第二大的数。

具体代码如下:

; Exercise 1.3
; Computer Square
(define (square x)
 (* x x))

; Computer sum of Squares
(define (sum_of_square x y)
 (+ (square x) (square y)))

; Find the bigger one between two numbers
(define (bigger x y)
  (if (> x y)
      x
      y))

; Find the smaller one between two numbers
(define (smaller x y)
  (if (< x y)
      x
      y))

; Computer the result
(define (square_sum_bigger_two x y z)
  (sum_of_square (bigger x y) (bigger (smaller x y) z)))

测试:

\> (square_sum_bigger_two 2 3 5)
\> 34

Exercise 1.8

Newton’s method for cube roots is based on the fact that if y is an approximation to the cube root of x, then a better approximation is given by the value

$(x/y^2+2y)/3 $

Use this formula to implement a cube-root procedure analogous to the square-root procedure.

解答:

这道题答案的结构与例题1.1.7很相似，需要注意的细节是与立方相关的计算，请具体参考下面的代码：

; Cube-root

; Compute cube
(define (cube x)
  (* x x x))

; Compute square
(define (square x)
  (* x x))

; computation strategy
(define (cube_root guess x)
  (if (good-enough_cube? guess x)
      guess
      (cube_root (improve_cube guess x) x)))

; Compute a better approximation
(define (improve_cube guess x)
  (/ (+ (/ x (square guess)) (* 2 guess)) 3))

; Test the guess is good enough?
(define (good-enough_cube? guess x)
  (< (abs (- (cube guess) x)) 0.001 ))

; Main function to compute cube root
(define (cube_r x)
  (cube_root 1.0 x))

测试结果如下:

\> (cube_r 31.00614)
\> 3.1415926337132434

Exercise 1.9

这里将第一个+替换为plus, 以下是第一个plus:

(define (plus a b)
 (if (= a 0)
     b
     (inc (plus (dec a) b))))

下面是对(plus 4 5)进行求值的具体步骤:

(plus 4 5)
|(inc (plus 3 5))
|(inc (inc (plus 2 5)))
|(inc (inc (inc (plus 1 5))))
|(inc (inc (inc (inc (plus 0 5)))))
|(inc (inc (inc (inc 5)))
|(inc (inc (inc 6)))
|(inc (inc 7))
|(inc 8)
9

很明显，这是一个线性递归计算过程(a linear recursive process)。

下面是第二个+的定义:

(define (+ a b)
 (if (= a 0)
     b
     (+ (dec a) (inc b))))

以下是对(+ 4 5)进行求值的具体过程:

(+ 4 5)
|(+ 3 6)
|(+ 2 7)
|(+ 1 8)
|(+ 0 9)
9

这是一个线性迭代计算过程(a linear iterative process)。

Exercise 1.10

这道题是有关Ackermann函数的计算，此方程的定义如下:

(define (A x y)
 (cond ((= y 0) 0)
       ((= x 0) (* 2 y))
       ((= y 1) 2)
       (else (A (- x 1)
                (A x (- y 1))))))

以下是对几个不同的x,y进行求值的结果:

]=> (A 1 10)
]=> 1024

]=> (A 2 4)
]=> 65536

]=> (A 3 3)
]=> 65536

现在我们看下以下几个计算过程:
第一个是函数f:

(define (f n) (A 0 n))
n 0 1 2 3 …
v 0 2 4 6 …

通过观察，这个函数定义的计算过程是 $f=2*n$。

第二个函数是g:

(define (g n) (A 1 n))
n 0 1 2 3 4 …
v 0 2 4 8 16 …

通过观察，这个函数定义的函数是 $g=2^n$

第二个函数是h:

(define (h n) (A 2 n))
n 0 1 2 3 4 …
v 0 2 4 16 65563 …

通过观察，这个函数定义的函数是 $h=2^(2^(.^(.^2)))}n$(指数是从上向下计算)。此时, Ackermann函数的增长速度非常快，当y=5时已无法计算出具体数值, 请注意这还仅仅是x=2。

另外值得一提的是,本题对Ackermann函数的定义与常见的形式有所不同, 下面给出更加常见的形式:

(define (Ack x y)
 (cond ((= x 0) (+ y 1))
      ((= y 0) (Ack (- x 1) 1))
      (else (Ack (- x 1)
                 (Ack x (- y 1))))))

如果有兴趣, 有关Ackermann函数的更多性质可以参见wiki页面。
